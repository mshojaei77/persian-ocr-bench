# پیام‌های پین تاپیک‌ها + پیشنهاد تاپیک‌های جدید (Bina OCR)

این فایل شامل پیام‌های پین آماده برای هر تاپیک تلگرام، فرمت یکپارچه ارسال داده، و پیشنهادهایی برای تاپیک‌های جدید است.
سبک نگارش با لحن جامعه (Telegram) حفظ شده تا مستقیماً قابل کپی باشد.

---

## تاپیک‌های فعلی و پیام پین پیشنهادی

### ۱. General (عمومی)
```
این تاپیک برای گفتگوهای عمومی پروژه است.

موارد مناسب:
• سؤال‌های کلی
• پیشنهادها
• معرفی اعضا
• هماهنگی همکاری
• ایده‌های جدید
• بحث‌هایی که تاپیک تخصصی ندارند

برای داده، مدل، لیبل‌گذاری و بنچمارک لطفاً از تاپیک مربوط استفاده کنید.

🤝 راه‌های مشارکت (از آسان به پیشرفته):
۱. ارسال داده خام (عکس/اسکن — نیاز به لیبل نیست)
۲. ارسال داده لیبل‌شده (بهترین نوع مشارکت)
۳. کمک به لیبل‌گذاری و بازبینی (بسیار مورد نیاز)
۴. معرفی مدل/دیتاست جدید
۵. ارسال Pull Request و مشارکت در Discussionها

هدف پروژه: ساخت OCR متن‌باز و دقیق برای اسناد فارسی چاپی و دست‌نویس
همراه با دیتاست، بنچمارک، لیدربورد و مدل‌های منتشرشده در GitHub و Hugging Face.

تمرکز فعلی: فاز ۱ — جمع‌آوری داده و انتخاب بهترین مدل‌های پایه.
```

### ۲. Data: Handwritten (داده دست‌نویس)
```
این تاپیک مخصوص جمع‌آوری دست‌خط فارسی است.

نمونه‌های مناسب:
• یادداشت روزمره
• جزوه و دفتر
• فرم دست‌نویس
• نامه
• آدرس
• پاسخ امتحان
• دست‌خط رسمی و محاوره‌ای
• دست‌خط تمیز، سخت‌خوان و کم‌کیفیت

در صورت وجود، متن صحیح هر تصویر را هم همراه آن ارسال کنید.
اگر متن استخراج‌شده را ندارید، خود تصویر خام هم کافی و مفید است.

ارسال داده لیبل‌شده: لطفاً به تاپیک «Data: Already Labeled» مراجعه کنید.
```

### ۳. Data: Typed (داده چاپی)
```
این تاپیک مخصوص جمع‌آوری داده‌های چاپی/تایپ‌شده فارسی است.

نمونه‌های مناسب:
• کتاب، مقاله و روزنامه
• فرم، نامه و اسناد اداری
• فاکتور، رسید و قبض
• تصاویر موبایلی از متن چاپی
• متن‌های چندستونه
• جداول
• متن ترکیبی فارسی و انگلیسی

در صورت وجود، متن صحیح هر سند را هم همراه آن ارسال کنید.
ارسال داده لیبل‌شده: لطفاً به تاپیک «Data: Already Labeled» مراجعه کنید.
```

### ۴. Labeling (لیبل‌گذاری)
```
این تاپیک مخصوص فرایند لیبل‌گذاری داده‌هاست.

موارد مرتبط:
• تولید متن مرجع با مدل معلم
• بازبینی و اصلاح انسانی
• قوانین نگارشی و نرمال‌سازی فارسی
• بررسی اختلاف بین لیبل‌ها
• ثبت نمونه‌های مبهم یا دشوار

نکات کلیدی لیبل‌گذاری فارسی:
• متن را دقیقاً مطابق سند بنویسید؛ بدون ترجمه یا اصلاح املا.
• ی/ي/ى و ک/ك را حفظ کنید (در لایه Strict متمایزند).
• نیم‌فاصله (ZWNJ) بخشی از املای فارسی است؛ پاک نکنید.
• اعداد فارسی (۱۲۳)، عربی‑اندیک (١٢٣) و لاتین (123) متفاوت‌اند — همان‌چیزی که در سند هست را بنویسید.
• در متن مخلوط فارسی/انگلیسی، ترتیب خواندن (RTL/LTR) را رعایت کنید.
• در ویرایشگر از راست‌چین صحیح استفاده کنید (نوت‌پد ویندوز گاهی عدد لاتین را فارسی نشان می‌دهد).

سخت‌ترین بخش پروژه همین لیبل‌زدن است — هر کمکی حتی روی ۱۰–۲۰ نمونه، تأثیر بزرگی دارد.
```

### ۵. Data: Already Labeled (داده لیبل‌شده) — پین نهایی
```
📌 Data: Already Labeled

این تاپیک مخصوص ارسال داده‌هایی است که از قبل لیبل شده‌اند.

لطفاً داده‌ها را به‌صورت فایل .zip یا .rar ارسال کنید و داخل آرشیو،
فایل اصلی و لیبل آن دقیقاً نام یکسان داشته باشند.

فرمت مورد قبول:
• فایل اصلی: تصویر، سند یا PDF
• فایل لیبل: فقط .txt یا .md
• کدگذاری متن: ترجیحاً UTF-8
• نام فایل لیبل باید دقیقاً با نام فایل اصلی یکی باشد.

مثال:
data_1.png
data_1.txt

یا:

data_2.pdf
data_2.md

ساختار نمونه آرشیو:
dataset.zip
├── data_1.png
├── data_1.txt
├── data_2.jpg
├── data_2.md
├── data_3.pdf
└── data_3.txt

قوانین مهم:
• هر فایل اصلی باید یک فایل لیبل متناظر داشته باشد.
• لیبل‌ها فقط در قالب .txt یا .md ارسال شوند.
• از نام‌هایی مانند final.txt یا label1.txt استفاده نکنید.
• متن لیبل باید دقیقاً مطابق محتوای سند باشد؛
  بدون توضیح اضافی، ترجمه یا اصلاح املای متن اصلی.
• در صورت وجود چند صفحه، هر صفحه را جداگانه نام‌گذاری کنید:
  book_001.png + book_001.txt
  book_002.png + book_002.txt

همراه فایل، لطفاً نوع داده، تعداد نمونه‌ها و توضیح کوتاهی
درباره منبع یا روش لیبل‌گذاری بنویسید.

در ارسال داده، لطفاً اطلاعات شخصی، محرمانه و قابل‌شناسایی را حذف یا محو کنید.
```

### ۶. Models (مدل‌ها)
```
این تاپیک مخصوص بررسی و انتخاب مدل‌های کاندید است.

موارد مرتبط:
• معرفی مدل OCR یا VLM جدید
• لینک GitHub و Hugging Face
• اندازه مدل و نوع معماری
• پشتیبانی از فارسی
• نتایج اولیه روی داده‌های فارسی

هدف این فاز: بنچمارک ۵۰+ مدل و انتخاب ۳ تا ۵ مدل برتر
بر اساس دقت، نوع مدل و اندازه آن.

لیست کامل کاندیدها و برآورد دقت در فایل candidates.csv پروژه نگهداری می‌شود.
مدل‌های از پیش فاین‌تیون‌شده فارسی (مثل Qwen3-VL-2B Persian-Arabic OCR)
برای بنچمارک اولویت دارند.
```

## راهنمای یکپارچه ارسال داده (Unified Submission Rule)

برای همه تاپیک‌های داده اعمال شود:
1. فرمت لیبل همیشه **`.txt` یا `.md`** باشد (یکپارچه‌سازی).
2. فایل اصلی و لیبل **هم‌نام** باشند.
3. ارسال به صورت **ZIP یا RAR** (برای داده لیبل‌شده الزامی).
4. اطلاعات شخصی/محرمانه حذف یا محو شود.
5. همراه ارسال: نوع داده + تعداد + منبع/روش لیبل‌گذاری.
